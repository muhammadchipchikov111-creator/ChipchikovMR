Контрольные вопросы по Git и системам контроля версий

1. Что такое система контроля версий?

Система контроля версий (Version Control System, VCS) - это инструмент, который позволяет разработчикам отслеживать изменения в коде и управлять ими. Она позволяет сохранять историю изменений, работать в команде, разрешать конфликты и восстанавливать предыдущие версии кода при необходимости.

2. Что такое Git?

Git - это распределенная система управления версиями. Он позволяет отслеживать изменения в файлах и координировать работу нескольких разработчиков над одним проектом.

3. Какие преимущества предоставляет Git?

Git предоставляет множество преимуществ, таких как возможность работы в команде, отслеживание изменений, создание веток для параллельной разработки, возможность восстановления предыдущих версий кода и многое другое.

4. Для чего используется команда git init?

Команда `git init` используется для инициализации нового репозитория Git в текущей директории.

5. Чем Git отличается от GitHub?

Git - это система управления версиями, а GitHub - это веб-платформа для хостинга репозиториев Git. Git позволяет управлять версиями кода локально, тогда как GitHub предоставляет возможность совместной работы над проектами, веб-интерфейс для управления репозиториями и дополнительные функции, такие как issue tracking и pull requests.
